## US-006.1: View Panel Member List
**As a** Master Admin or Practice Admin  
**I want to** view all panel members in the system  
**So that** I can see available interviewers and their qualifications

### Acceptance Criteria:
- [ ] Display panel members in a table with columns: Name, Role, Level, Practice, Qualification Status, Actions
- [ ] Master Admin sees all panel members across all practices
- [ ] Practice Admin sees panel members in their practice only
- [ ] Display panel member count
- [ ] Show active and inactive members with status badges
- [ ] Display qualification indicators (qualified/not qualified for specific levels)
- [ ] Default sorting by name (ascending)
- [ ] Empty state message when no panel members exist

### UI Elements:
- Panel members table with sortable columns
- Member count badge
- Status indicator (active/inactive)
- Qualification badges (levels they can interview for)
- Action menu for each row
- "Add Panel Member" button

### Business Rules:
- Panel members are users with "Tech Panel Member" role
- Each panel member can be qualified for multiple designation levels
- Master Admin: View all panel members
- Practice Admin: View panel members in assigned practice

---

## US-006.2: Add New Panel Member
**As a** Master Admin or Practice Admin  
**I want to** add a new panel member to the system  
**So that** they can conduct interviews

### Acceptance Criteria:
- [ ] "Add Panel Member" button opens create panel member form
- [ ] Form displays all required fields with validation
- [ ] Username field validates Active Directory format (min 5, max 100 characters)
- [ ] First Name and Last Name fields (min 2, max 50 characters each)
- [ ] Email field validates aspiresys.com domain and uniqueness
- [ ] Phone number field with country code (optional)
- [ ] Role automatically set to "Tech Panel Member"
- [ ] Practice dropdown populated with active practices
- [ ] Practice field is mandatory
- [ ] Master Admin can select any practice; Practice Admin can only assign to their practice
- [ ] Qualification section allows selecting designation levels member can interview for
- [ ] Multi-select for applicable levels (e.g., Junior, Mid-Level, Senior)
- [ ] Source dropdown (Web, Mobile, API, Admin)
- [ ] Password is auto-generated by system
- [ ] "Create" button creates panel member and displays success message
- [ ] "Cancel" button closes form without saving
- [ ] Display validation errors inline
- [ ] Success notification shows created member details

### UI Elements:
- "Add Panel Member" button (primary action)
- Modal/page form with fields:
  - UserName (text input, required)
  - FirstName (text input, required)
  - LastName (text input, required)
  - EmailID (email input, required)
  - CountryCode (dropdown, optional)
  - PhoneNumber (text input, optional)
  - RoleID (read-only, set to "Tech Panel Member")
  - PracticeID (dropdown, required)
  - Applicable Levels (multi-select checkboxes, required)
  - Source (dropdown, required)
- Create button (primary)
- Cancel button (secondary)
- Inline validation messages
- Success toast notification

### Business Rules:
- Panel member must be qualified for at least one designation level
- Username, Email, and Phone must be unique
- Panel members are essentially users with specific role and qualifications
- System generates MemberPanelApplicabilityID for level mappings
- Welcome email sent with credentials

## US-006.3: View Panel Member Details
**As a** Master Admin or Practice Admin  
**I want to** view detailed information about a panel member  
**So that** I can see their qualifications and interview history

### Acceptance Criteria:
- [ ] Clicking member name or "View" action opens detail modal/page
- [ ] Display all member information: Name, Email, Phone, Practice, Role
- [ ] Show qualification levels with visual indicators
- [ ] Display interview statistics:
  - Total interviews conducted
  - Interviews this month
  - Average rating given (if applicable)
  - Upcoming interviews scheduled
- [ ] Show current availability status
- [ ] Display audit information: Created Date, Modified Date
- [ ] "Edit" and "Deactivate" buttons available (if permissions allow)
- [ ] "Manage Qualifications" button to update applicable levels
- [ ] "View Schedule" button to see member's interview calendar
- [ ] "Close" button returns to member list

### UI Elements:
- Panel member detail modal/page
- Information sections: Personal Info, Practice & Role, Qualifications, Interview Statistics, Availability, Audit Info
- Action buttons: Edit, Deactivate, Manage Qualifications, View Schedule, Close
- Qualification badges for each level
- Statistics cards

## US-006.4: Edit Panel Member Details
**As a** Master Admin or Practice Admin  
**I want to** edit panel member information  
**So that** I can update member details when needed

### Acceptance Criteria:
- [ ] "Edit" action opens edit form with pre-filled data
- [ ] Allow editing: FirstName, LastName, EmailID, PhoneNumber, PracticeID
- [ ] Username cannot be edited (read-only)
- [ ] Role remains "Tech Panel Member" (read-only)
- [ ] All validations apply as in Add form
- [ ] Practice Admin can only edit members in their practice
- [ ] Master Admin can edit any panel member
- [ ] "Save" button updates member and displays success message
- [ ] "Cancel" button discards changes
- [ ] Audit fields updated automatically (ModifiedDate, ModUser)

### UI Elements:
- Edit form modal with pre-filled fields
- Save button (primary)
- Cancel button (secondary)
- Success notification

### Business Rules:
- Email uniqueness validated on update
- Phone number uniqueness validated if changed
- Cannot edit deactivated panel members

## US-006.5: Manage Panel Member Qualifications
**As a** Master Admin or Practice Admin  
**I want to** update panel member qualifications  
**So that** they can interview candidates at appropriate levels

### Acceptance Criteria:
- [ ] "Manage Qualifications" button opens qualification management modal
- [ ] Display current qualification levels with checkboxes
- [ ] Checkboxes for each designation level: Junior, Mid-Level, Senior, Lead
- [ ] Check/uncheck to add/remove qualifications
- [ ] At least one level must be selected
- [ ] Display designation titles associated with each level (if applicable)
- [ ] "Save Changes" button updates qualifications
- [ ] Success message confirms update
- [ ] Updated qualifications reflected in member list and details

### UI Elements:
- Manage Qualifications modal
- Checkbox list for designation levels
- Associated designation titles display
- Save Changes button (primary)
- Cancel button (secondary)
- Validation message for minimum selection
- Success notification

### Business Rules:
- Panel member must have at least one qualification level
- Qualification changes affect member's availability for interviews
- MemberPanelApplicabilityID records are created/updated
- Cannot remove qualification if member has upcoming interviews for that level (validation)

## US-006.6: Deactivate Panel Member
**As a** Master Admin or Practice Admin  
**I want to** deactivate a panel member  
**So that** they no longer appear in interviewer selection lists

### Acceptance Criteria:
- [ ] "Deactivate" action available in member row or detail view
- [ ] Clicking "Deactivate" displays confirmation dialog
- [ ] Confirmation explains consequences (cannot conduct new interviews, removed from dropdowns)
- [ ] Check for upcoming scheduled interviews
- [ ] If upcoming interviews exist, show warning and list of affected interviews
- [ ] User must confirm deactivation
- [ ] Upon confirmation, member's IsActive status set to false
- [ ] Deactivated member shown with "Inactive" badge in list
- [ ] Success message displayed after deactivation
- [ ] Deactivated members cannot log in

### UI Elements:
- Deactivate button
- Confirmation dialog with warning
- List of affected interviews (if any)
- Inactive status badge
- Success notification

### Business Rules:
- Cannot deactivate member with upcoming interviews (must reschedule first)
- Practice Admin can only deactivate members in their practice
- Master Admin can deactivate any panel member
- Deactivation is soft delete (IsActive = false)
- Deactivated members still visible in list with filter

### Related FR: FR#MAP-5 – Deactivate Member

## US-006.7: Reactivate Panel Member
**As a** Master Admin or Practice Admin  
**I want to** reactivate a previously deactivated panel member  
**So that** they can conduct interviews again

### Acceptance Criteria:
- [ ] "Reactivate" button available for inactive panel members
- [ ] Clicking "Reactivate" displays confirmation dialog
- [ ] Upon confirmation, member's IsActive status set to true
- [ ] Reactivated member appears as active in list
- [ ] Success message displayed after reactivation
- [ ] Member can log in and conduct interviews again
- [ ] Previous qualifications remain intact

### UI Elements:
- Reactivate button (for inactive members)
- Confirmation dialog
- Success notification

---

## US-006.8: Search and Filter Panel Members
**As a** Master Admin or Practice Admin  
**I want to** search and filter panel members  
**So that** I can quickly find specific interviewers

### Acceptance Criteria:
- [ ] Search box allows searching by name, email, or username
- [ ] Search results update as user types (debounced)
- [ ] Filter by practice (dropdown) - for Master Admin
- [ ] Filter by qualification level (multi-select: Junior, Mid-Level, Senior, Lead)
- [ ] Filter by availability status (Available, Unavailable, All)
- [ ] Filter by status (Active, Inactive, All)
- [ ] Multiple filters can be applied simultaneously
- [ ] "Clear Filters" button resets all filters
- [ ] Display count of filtered results

### UI Elements:
- Search input field
- Practice filter dropdown (Master Admin only)
- Qualification level multi-select
- Availability filter dropdown
- Status filter dropdown
- Clear Filters button
- Results count label

## US-006.9: View Panel Member Availability
**As a** Master Admin or Practice Admin  
**I want to** see panel member availability status  
**So that** I know who is available for scheduling interviews

### Acceptance Criteria:
- [ ] Availability indicator displayed in member list (Available/Busy/Unavailable)
- [ ] Color-coded badges for quick identification
- [ ] In detail view, show detailed availability:
  - Upcoming interviews count
  - Available slots this week
  - Blocked/unavailable periods

- [ ] Real-time or near-real-time updates

- Availability details section
- Mini calendar view with interview slots
- Link to full schedule view

## US-006.10: View Panel Member Interview History
**As a** Master Admin or Practice Admin  
**I want to** view a panel member's interview history  
**So that** I can track their interview activity and performance

### Acceptance Criteria:
- [ ] "Interview History" button/tab in member detail view
- [ ] Display interviews in chronological order (most recent first)
- [ ] Show interview details: Candidate Name, Date/Time, Designation Level, Feedback Status

## US-006.11: Assign Panel Member to Practice
**As a** Master Admin  
**I want to** reassign a panel member to a different practice  
**So that** resources can be reallocated as needed

### Acceptance Criteria:
- [ ] "Change Practice" action available (Master Admin only)
- [ ] Opens practice selection modal
- [ ] Display current practice
- [ ] Dropdown to select new practice
- [ ] Warning message about impact (scheduled interviews, qualification mappings)
- [ ] Confirmation required
- [ ] Upon confirmation, practice assignment updated
- [ ] Success message confirms change

### UI Elements:
- Change Practice button (Master Admin only)
- Practice selection modal
- Current practice display
- New practice dropdown
- Warning message
- Confirm and Cancel buttons
- Success notification

### Business Rules:
- Only Master Admin can change practice assignments
- Check for upcoming interviews in current practice
- Qualification mappings remain but scoped to new practice

## US-006.12: Sort and Pagination
**As a** Master Admin or Practice Admin  
**I want to** sort and paginate panel member list  
**So that** I can navigate large lists efficiently

### Acceptance Criteria:
- [ ] Sortable columns: Name, Email, Practice, Qualification Levels
- [ ] Click column header to sort ascending, click again for descending
- [ ] Sort indicator displayed
- [ ] Pagination controls at bottom
- [ ] Page size options: 10, 20, 50
- [ ] Default: 20 members per page
- [ ] Total count display

### UI Elements:
- Sort indicators in column headers
- Pagination controls
- Page size dropdown
